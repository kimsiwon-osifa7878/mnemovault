# MnemoVault 퀵 스타트 가이드

> LLM이 지식을 컴파일하여 영구적으로 축적하는 위키 IDE, MnemoVault를 빠르게 시작해보세요.

---

## 1. 설치 및 실행

```bash
# 프로젝트 설치
npm install

# 개발 서버 실행
npm run dev
```

브라우저에서 **http://localhost:3000** 을 열면 MnemoVault가 실행됩니다.

---

## 2. 화면 구성 한눈에 보기

MnemoVault는 3개의 패널로 구성된 IDE 스타일의 레이아웃입니다.

```
┌──────────────┬─────────────────────────┬──────────────┐
│              │                         │              │
│  ① 사이드바   │    ② 에디터/프리뷰        │  ③ 우측 패널  │
│              │                         │  (Graph/Chat)│
│  파일 트리     │    위키 문서 편집/보기      │              │
│  검색         │                         │  그래프 뷰    │
│  새 페이지     │                         │  채팅        │
│  Ingest      │                         │  Lint        │
│  LLM 설정     │                         │  설정        │
│              │                         │              │
└──────────────┴─────────────────────────┴──────────────┘
```

---

## 3. 각 패널 상세 설명

### ① 사이드바 (왼쪽)

사이드바는 위키의 모든 페이지를 탐색하고 관리하는 공간입니다.

#### MnemoVault 로고
- 최상단에 앱 이름이 표시됩니다.

#### 🔍 검색
- **"Search pages..."** 입력란에 키워드를 입력하면 페이지를 실시간으로 필터링합니다.

#### 특수 페이지
- **Wiki Index** — 전체 위키의 카탈로그입니다. 모든 페이지가 카테고리별로 정리되어 있습니다.
- **Wiki Log** — 시간순으로 모든 작업 기록이 기록됩니다 (Ingest, Query 등).

#### 📁 파일 트리
페이지들이 4가지 타입별로 분류되어 표시됩니다:
- **Concepts** — 개념 페이지 (예: 트랜스포머 아키텍처)
- **Entities** — 엔티티 페이지 (예: 인물, 조직)
- **Sources** — 소스 요약 페이지 (Ingest로 생성됨)
- **Analyses** — 분석 페이지 (Query 답변을 저장한 것)

각 섹션의 ▶ 화살표를 클릭하면 펼치거나 접을 수 있습니다.

#### 하단 버튼들

| 버튼 | 설명 |
|------|------|
| 🟢/🔵 **LLM 상태 표시** | 현재 사용 중인 LLM을 보여줍니다. 클릭하면 LLM 설정 열기 |
| ＋ **New Page** | 새 위키 페이지를 만듭니다 |
| ⬆ **Ingest Source** | 소스 파일을 위키로 컴파일합니다 |

---

### ② 에디터/프리뷰 (중앙)

위키 문서를 보고 편집하는 핵심 영역입니다.

#### 상단 툴바

왼쪽에는 현재 열린 문서의 **제목**과 **타입 뱃지**(CONCEPT, ENTITY 등)가 표시됩니다.
수정 사항이 있으면 제목 옆에 🟠 주황색 점이 나타납니다.

오른쪽에는 다음 버튼들이 있습니다:

| 버튼 | 아이콘 | 설명 |
|------|--------|------|
| **Edit** | ✏️ | 마크다운 에디터 모드 (직접 편집) |
| **Preview** | 👁 | 렌더링된 문서 보기 모드 |
| **Split** | ▥ | 왼쪽 편집 + 오른쪽 프리뷰 동시 보기 |
| **Save** | 💾 | 변경사항 저장 (`Ctrl+S` / `Cmd+S`) |
| **Delete** | 🗑 | 현재 페이지 삭제 (확인 메시지 표시) |

#### `[[위키링크]]` 표시 방식
- **파란색** 링크 → 해당 페이지가 위키에 존재함. 클릭하면 해당 페이지로 이동.
- **빨간색** 링크 → 아직 만들어지지 않은 페이지.

#### Backlinks 섹션
문서 하단에 **이 페이지를 참조하는 다른 문서 목록**이 표시됩니다. 클릭하면 해당 문서로 이동합니다.

---

### ③ 우측 패널 (오른쪽)

상단 탭으로 **Graph**와 **Chat** 두 가지 뷰를 전환할 수 있습니다.

#### Graph 뷰
위키 페이지 간의 `[[위키링크]]` 관계를 **Force-directed 그래프**로 시각화합니다.

**노드 색상:**
| 색상 | 타입 |
|------|------|
| 🔵 파랑 | Concept (개념) |
| 🟣 보라 | Entity (엔티티) |
| 🟢 초록 | Source (소스) |
| 🟠 주황 | Analysis (분석) |
| ⚪ 흰색 | 현재 선택된 노드 |

노드를 **클릭**하면 해당 위키 페이지가 에디터에 열립니다.

#### Chat 뷰
위키 지식을 기반으로 LLM에게 질문하는 **Context-Aware 채팅**입니다.

**특징:**
- 현재 열린 문서가 자동으로 컨텍스트에 포함됩니다 (상단에 "Context: 페이지명" 표시)
- 답변에 포함된 `[[위키링크]]`를 클릭하면 해당 페이지로 이동합니다
- **"Save to wiki"** 버튼을 켜면 답변이 Analysis 페이지로 위키에 저장됩니다

#### 패널 상단 버튼
| 버튼 | 설명 |
|------|------|
| ⚠️ (삼각형) | **Lint** — 위키 헬스체크 실행 |
| ⚙️ (톱니바퀴) | **LLM Settings** — LLM 프로바이더 설정 |

#### 패널 숨기기
화면 오른쪽 하단의 패널 버튼으로 우측 패널을 접거나 펼 수 있습니다.

## 4. 핵심 기능 사용법

### 🔧 LLM 설정하기 (가장 먼저!)

MnemoVault를 사용하려면 먼저 LLM을 설정해야 합니다.

**방법:** 사이드바 하단의 LLM 상태 표시 버튼 또는 우측 패널의 ⚙️ 버튼 클릭

#### Claude (Anthropic API) 사용 시
1. Provider에서 **Claude** 선택
2. 원하는 모델 선택 (Claude Sonnet 4 / Haiku 4.5 / Opus 4.6)
3. 환경 변수에 API 키 설정:
   ```env
   ANTHROPIC_API_KEY=your-api-key-here
   ```

#### Ollama (로컬 LLM) 사용 시
1. Provider에서 **Ollama** 선택
2. Ollama URL 입력 (기본: `http://localhost:11434`)
3. **Connect** 버튼 클릭
4. ✅ "Connected — N model(s) found" 메시지가 나타나면 성공
5. 드롭다운에서 원하는 모델 선택
6. **Done** 클릭

> 💡 Ollama는 API 키가 필요 없습니다! 로컬에서 무료로 사용할 수 있습니다.
>
> ```bash
> # Ollama 설치 후
> ollama serve          # Ollama 서버 시작
> ollama pull llama3    # 모델 다운로드
> ```

---

### 📥 Ingest — 소스를 위키로 컴파일하기

Ingest는 MnemoVault의 핵심 기능입니다. 원본 소스(논문, 기사, 노트)를 LLM이 분석하여 구조화된 위키 페이지로 변환합니다.

**사용법:**
1. 사이드바 하단의 **"Ingest Source"** 버튼 클릭
2. 파일을 **드래그&드롭** 하거나 **browse**를 클릭하여 파일 선택 (.md, .txt, .pdf, .json 지원)
3. **Source Type** 선택: `article` | `paper` | `note` | `data`
4. **Ingest** 버튼 클릭
5. LLM이 소스를 분석하여 자동으로 생성:
   - 📄 소스 요약 페이지 (Sources 폴더)
   - 💡 개념 페이지들 (Concepts 폴더)
   - 👤 엔티티 페이지들 (Entities 폴더)
   - 🔗 자동 교차참조 `[[위키링크]]`
   - 📋 index.md와 log.md 자동 갱신

**결과 화면:**
- ✅ "Ingest Complete" — 생성된 페이지 목록이 표시됩니다
- 사이드바에 새 페이지들이 자동으로 나타납니다

---

### 💬 Query — 위키에 질문하기

Chat 뷰에서 위키 지식을 기반으로 LLM에게 질문할 수 있습니다.

**사용법:**
1. 우측 패널에서 **Chat** 탭 선택
2. 에디터에서 관련 문서를 열어두면 자동으로 컨텍스트에 포함됨
3. 하단 입력란에 질문 입력 후 **전송** 버튼 클릭 (또는 Enter)
4. LLM이 위키 내용을 참조하여 답변

**답변을 위키에 저장하기:**
- 질문 전에 **"Save to wiki"** 버튼을 켜두면 답변이 Analyses 폴더에 자동 저장됩니다

---

### 🔍 Lint — 위키 건강 검사

위키의 문제점을 자동으로 발견합니다.

**사용법:**
1. 우측 패널 상단의 ⚠️ 버튼 클릭
2. **"Run Lint"** 버튼 클릭
3. 발견된 이슈들이 타입별로 표시됨:

| 타입 | 색상 | 설명 |
|------|------|------|
| **orphan** | 🟡 노랑 | 다른 페이지에서 참조되지 않는 고아 페이지 |
| **missing_page** | 🟣 보라 | `[[위키링크]]`가 존재하지 않는 페이지를 가리킴 |
| **contradiction** | 🔴 빨강 | 페이지 간 내용이 모순됨 (LLM이 감지) |
| **stale** | 🟠 주황 | 오래되어 업데이트가 필요한 내용 |
| **missing_crossref** | 🔵 파랑 | 누락된 교차참조 |

---

### 📝 페이지 직접 만들기

**사용법:**
1. 사이드바 하단의 **"New Page"** 버튼 클릭
2. **Title** 입력 (예: "트랜스포머 아키텍처")
3. **Type** 선택: `concept` | `entity` | `source` | `analysis`
4. **Create** 버튼 클릭 (또는 Enter)
5. 새 페이지가 에디터에 열림 → 마크다운으로 내용 작성

**편집 팁:**
- 다른 페이지를 참조할 때는 `[[페이지 제목]]` 형식으로 위키링크를 작성하세요
- `Ctrl+S` (또는 `Cmd+S`)로 빠르게 저장할 수 있습니다

## 5. 추천 워크플로우

처음 사용한다면 이 순서를 따라해보세요:

```
1️⃣  LLM 설정     →  Ollama 또는 Claude 연결
2️⃣  Ingest       →  관심 있는 기사/논문을 하나 넣어보기
3️⃣  탐색          →  자동 생성된 페이지들을 Graph 뷰에서 확인
4️⃣  Query        →  Chat에서 위키에 질문해보기
5️⃣  편집          →  생성된 페이지를 수정하거나 새 페이지 추가
6️⃣  Lint         →  위키가 좀 쌓이면 Lint로 건강 체크
7️⃣  반복          →  소스를 더 추가하며 위키를 성장시키기
```

> 💡 **핵심 철학:** 소스를 넣을수록 위키는 복리로 성장합니다.
> 교차참조가 자동으로 연결되고, 기존 페이지가 보강되며, 지식의 네트워크가 풍부해집니다.

## 7. 문제 해결

### Ollama에 연결할 수 없어요
- `ollama serve` 명령으로 Ollama가 실행 중인지 확인하세요.
- URL이 올바른지 확인하세요 (기본: `http://localhost:11434`).
- 모델이 다운로드되어 있는지 확인하세요 (`ollama list`).

### Ingest 결과가 이상해요
- 소스 파일이 텍스트 형식인지 확인하세요 (.md, .txt 권장).
- LLM 모델을 더 큰 것으로 변경해보세요 (예: Claude Opus, llama3:70b).

### 페이지가 사이드바에 안 보여요
- 브라우저를 새로고침하면 페이지 목록이 갱신됩니다.
